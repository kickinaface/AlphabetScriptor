ALPHABET:








Font Formats


There are two formats of the script.

Traditional:





*A line format that is the first creation of this alphabet. This adaptation produces an effect that allows the learner to engage deeply with their intentions.

Print/block default:




*A block print format that removes the line and renders the alphabet characters only. This method is quick powerful and the first to be programmed.


*The picture in the top heading title above is the alphabet of this script and should be learned and memorized just as the learner understands the current alphabet.

This is to ensure the content is absorbed into the sub-conscious mind.

The ability to both read and write in this script or any script is imparitive before proceeding to computational typing or automated tools.

The traditional way of learning is the best way. In this particular case it is in alignment with previous writing styles that have already been acquired.

Once you are able to learn the script and read and write with it without reference material you are ready to express with it and truly embrace this tool as well.




Download Alphabet-Scriptor

Click Here to download a .zip file of the application. Please follow the documentation on how to install it on your system. This is a NodeJS application and it will be used via the browser.
You do not have to download in order to use this application. Although you must download if you wish to use the "Editor" program for your own purposes. Please continue reading more for setup.




Application Setup

After you have downloaded the .ZIP file and extracted it to your prefered location. You must have NodeJS installed on your machine.

Head over here to install it.


Run the App

Open up an instance of terminal or command line to run the NodeJS command. Make sure you have pointed the instance to the proper location of the application.

Run: "npm install" to install the dependencies at first.


Then run: "node main" to start the program




Head to your browser and type: "http://localhost:7000" into the url to visit the web application.




How to Use the Editor

Follow these instructions on how to use the Editor program.

When you know how to edit files and run the application, you can create specific pages or "passages" that are formatted to your liking.

Use cases of the Editor are for: longer size documents, web or content developers and the creation of pages that can be visited by themselves locally or on a server.

Change "index.html"


Once you have downloaded the application and you have it running locally, visit the: "Editor" program in the navigation or by visiting "/Editor" after your root url.

You will see the what is being loaded inside the index.html file within the "Editor" folder.
In your code editor of choice, locate and open the "index.html" file to make changes to it.










Begin making changes to the code. Everything will render that is kept inside a ".readerWrapper" class.
Try changing or removing some of the classes that hold the textual content.
Be sure to keep the ".translationWrapper" class. It is needed to render the new text by the Editor program.
You can un-comment out the image to see how images are added in. Follow this format to "edit" and create documents of your own.









Once you have "index.html" to your liking, try adding some basic features to it like: Dark or Light, text alignment, and font size.
You can change this in the UI or you can follow the code to program it in the controls so that they load up with your document's first launch.

Editor's Features

At the moment, there are 4 major features of Editor. More to come soon! Please follow along to take advantage of these features.




Reading Modes:

As of right now, there are two reading modes: Dark and Light, click the two to toggle between them.
It is said that dark backgrounds are best for reading while white background is best for printing.

Text Alignment:

You can align the text to the left, center, or right or the screen.

Text Size:

Change the text size of your document

Font Formats:

As of right now, you can change the font formats programmaticallly. Later, there where will be UI controls to toggle font formats.






*Take a look at the code for customizing the controls.
This is telling the page the be: "centered"
The Rendering Mode to be: "Dark"
The image width or size of the image: "17"
And lastly the font type or format which is: "default" also known as print or block form.
These settings were set so that the document loads with these values on page refresh without having to click on the UI controls.




How to Use the Type Writer

Follow these instructions on how to use the Type Writer.

I like to use Type Writer for short but powerful proverbs that I usually make dead centered and maybe with a black background.
Later, I like to print off each page individually or use screen shots of the text in Type Writer and then transfer the cropped
image onto something that I wish to print or work with later. I have specifically done this with gifts and also with events journals.
Its fun to see your affirmations printed out and placed onto things.

Controls:




Change Dark/Light

Change Text Alignment


Add Page

Click "Add (+)" to create a new page on the right panel.
Navigate to the new page on the control bar.
Add your new page content.
Remember to Save your content into memory.

Remove Page

Click "Remove (-)" to remove a page. Be sure you are on the present page you wish to remove.

Save Page

Click "Save" to hold the content in memory for each page.

Export Workspace

Click "Export" to export the current workspace. This includes the pages and their content. They will be exported as a ".json" document.
You can download this file and use it for later to be imported or just to save your work.

Import Workspace

Click "Import" to bring up the dialog box. Then, select the .json file which you have exported earlier.
Alphabet-Scriptor will not recognize any other format than the .json document that was exported previously.











Select "Import" button on the dialog to replace the current workspace with the imported one.
